    - Explicações sobre Branch e Merge

        Branch Master é a versão oficial do código ou projeto, podendo ser adicionados vários commits a ela, commits são melhoramentos ou versões do que são adicionados ao Branch Master, no entanto, pode-se também criar um Branch secundário, que seria o Branch Feature que também podemos adicionar commits a ele.

        Basicamente uma versão oficial (Branch Master e seus commits), e é criado um Branch secundário dele para uma nova versão do projeto maior e depois uni-los e tornar um só novamente.
